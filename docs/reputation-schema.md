# Reputation Schema Specification

**Version:** 1.0
**Status:** Canonical
**Defined by:** TODO-08
**Consumed by:** TODO-09, TODO-11

---

## Overview

A reputation file records a single NPC's computed opinion of the player. It is the output of the judgment system (`engine/judge_player.ts`), which aggregates an NPC's memories, values, and personality tendencies into a numeric score and a named disposition label. The file also contains an LLM-generated internal monologue — the NPC's private thoughts about the traveler — and a list of the key memories driving their opinion.

Reputation files are **recomputed**, not edited by hand. Each time the judgment system runs for an NPC, the previous reputation file is overwritten with the new calculation.

---

## File Location

```
world/town/npcs/<npc_id>/reputation/player.md
```

There is exactly one reputation file per NPC. It always describes the NPC's opinion of the player (not of other NPCs).

---

## Document Structure

A reputation file has two parts:

1. **YAML Frontmatter** — structured scores and metadata (between `---` fences)
2. **Body** — the NPC's internal monologue, followed by a key memories listing

---

## YAML Frontmatter Fields

| Field | Type | Description |
|---|---|---|
| `npc_id` | `string` | The NPC's unique identifier (e.g. `"brynn-ashford"`) |
| `last_updated` | `string` | Date the reputation was last recomputed. Format: `YYYY-MM-DD` |
| `opinion_score` | `float` | Composite opinion score, clamped to `[-1.0, 1.0]` |
| `disposition` | `enum` | Named label derived from `opinion_score`. One of: `adoring`, `friendly`, `neutral`, `wary`, `hostile`, `hated` |
| `direct_memories` | `float` | Score component from direct player memories (weighted average of valence) |
| `rumors_heard` | `float` | Score component from rumor-sourced player memories (capped at 30% influence) |
| `values_alignment` | `float` | Score component from how player actions align with the NPC's core values |
| `memory_modifier` | `float` | Score component from soul-based tendencies (grudge-holding, forgiveness, moral grain) |

### Frontmatter Example

```yaml
---
npc_id: "brynn-ashford"
last_updated: "0001-06-15"
opinion_score: 0.4233
disposition: friendly
direct_memories: 0.5714
rumors_heard: -0.12
values_alignment: 0.3333
memory_modifier: 0
---
```

---

## Body — Internal Monologue

The body of the file contains the NPC's private internal monologue about the traveler, generated by the LLM. This is 2-4 sentences written in the NPC's voice, grounded in specific memories and values — not generic sentiment.

Below the monologue is a `## Key Memories` section listing the top 3 memories (by weight) that most shaped the NPC's opinion.

### Body Example

```markdown
They fixed my shutter without a word. That counts for something. But I keep hearing
about what happened in the Ashfall valley, and Tomas doesn't lie about things like that.
I want to trust them, but I'm watching.

## Key Memories

- 0001-03-15_player-helped-repair.md (positive, weight: 0.6)
- 0001-05-10_heard-player-stole-from-merchant.md (negative, weight: 0.4)
- 0001-04-12_player-defended-village.md (positive, weight: 0.8)
```

---

## Disposition Labels

The `disposition` field is derived from `opinion_score` using fixed thresholds:

| Disposition | Score Range | Behavioral Meaning |
|---|---|---|
| `adoring` | `>= 0.75` | The NPC deeply trusts the player. Will offer help unprompted, share secrets, defend the player's reputation. May overlook minor wrongs. |
| `friendly` | `>= 0.4` | The NPC likes the player. Warm in conversation, willing to help when asked, gives benefit of the doubt. |
| `neutral` | `>= -0.15` | No strong opinion. Polite but guarded. Treats the player like any other stranger or acquaintance. |
| `wary` | `>= -0.4` | The NPC is suspicious. Short answers, reluctant to help, watches the player carefully. May reference negative rumors. |
| `hostile` | `>= -0.74` | The NPC actively dislikes the player. May refuse service, spread negative rumors, or confront the player about past wrongs. |
| `hated` | `< -0.74` | The NPC despises the player. Will avoid interaction, may act against the player's interests, could become a threat. Redemption is extremely difficult. |

---

## Scoring Algorithm

The composite `opinion_score` is computed from four independent components, each contributing a weighted portion to the final score.

### Component 1: Direct Memory Score (50% weight)

A weighted average of the emotional valence of all memories where `source === "direct"` and `subject === "player"`.

- Positive valence = `+1.0`, neutral = `0.0`, negative = `-1.0`
- Each memory's contribution is scaled by its `weight` field
- Faded memories contribute at 20% of their original weight
- Formula: `sum(effectiveWeight * valence) / sum(effectiveWeight)`
- Result range: `[-1.0, 1.0]`

### Component 2: Rumor Score (20% weight)

Same weighted-average formula as direct memories, but applied only to memories where `source === "rumor"` and `subject === "player"`.

- Raw score is capped at `[-0.3, 0.3]` to prevent rumor consensus from dominating the opinion
- Rumors are inherently less reliable — this cap ensures direct experience always outweighs hearsay

### Component 3: Values Alignment Score (30% weight)

Measures how well the player's actions (as recorded in memories) align with the NPC's `core.values[]`.

For each player memory, the system checks if the memory body contains keywords associated with any of the NPC's core values (using the same `VALUE_KEYWORDS` map as the drift engine).

**Priority weighting by value index:**

| Value Position | Weight Multiplier |
|---|---|
| Index 0 (top value) | 3x |
| Index 1 | 2x |
| Index 2+ | 1x |

- A positive memory matching a valued topic contributes positive alignment
- A negative memory matching a valued topic contributes negative alignment
- Only the highest-priority matching value per memory is counted
- Result: weighted average of `(valence * priorityWeight * memWeight) / (priorityWeight * memWeight)`

### Component 4: Soul Modifier (additive)

Adjustments based on the NPC's personality tendencies and moral grain. This modifier is added directly to the composite before clamping.

### Composite Formula

```
opinionScore = clamp(
  (directScore * 0.5) + (rumorScore * 0.2) + (valuesScore * 0.3) + modifier,
  -1.0, 1.0
)
```

---

## Soul Modifier Rules

The soul modifier adjusts the composite score based on personality traits that make certain NPCs more or less forgiving.

| Condition | Effect | Rationale |
|---|---|---|
| Tendency includes "grudge" | `modifier -= abs(min(directScore, 0)) * 0.4` | Grudge-holders amplify negative direct experiences by 40% |
| Tendency includes "forgiv" | `modifier += abs(min(directScore, 0)) * 0.2` | Forgiving NPCs reduce the impact of negative memories by 20% |
| `moral_grain === "ruthless"` | Reduces negative contribution from low-weight (`< 0.5`) direct memories proportionally | Ruthless NPCs dismiss mild slights — only significant wrongs register |
| `moral_grain === "selfless"` | `modifier -= abs(min(directScore, 0)) * 0.5` | Selfless NPCs are deeply affected by harm — negative experiences hit 50% harder |

**Note:** Multiple modifiers can stack. A selfless NPC with a grudge-holding tendency will be hit particularly hard by negative memories. A forgiving NPC who is also ruthless may largely shrug off minor wrongs.

---

## When Recomputation Happens

Reputation is recomputed in these situations:

- **Before dialogue:** When the player approaches an NPC for conversation, the judgment system runs to ensure the NPC's disposition reflects their latest memories.
- **After memory creation:** When a new memory about the player is written (direct or rumor), the affected NPC's reputation is recalculated.
- **During world advance:** At the end of a simulation step, all NPCs who received new player-related memories have their reputations updated.

Each recomputation fully overwrites the previous `reputation/player.md` file. There is no history — only the current opinion matters.

## Worked Example

Consider **Brynn Ashford**, the innkeeper, with the following soul configuration:

```
name: "Brynn Ashford"
temperament: cautious
occupation: innkeeper
core.values: ["community", "honesty", "family"]
core.moral_grain: pragmatic
lived.tendencies: ["guards their words"]
```

Brynn has three memories about the player:

| # | Date | Type | Valence | Weight | Source | Faded | Body (excerpt) |
|---|---|---|---|---|---|---|---|
| 1 | 0001-03-15 | interaction | positive | 0.6 | direct | false | "The stranger helped me repair my shutter..." |
| 2 | 0001-04-02 | interaction | negative | 0.7 | direct | false | "They lied about where they found the coin..." |
| 3 | 0001-05-10 | rumor | negative | 0.3 | rumor | false | "Tomas says the traveler stole from the merchant..." |

### Step 1: Direct Memory Score

Direct memories: #1 (positive, weight 0.6) and #2 (negative, weight 0.7).

```
weightedSum = (0.6 * 1.0) + (0.7 * -1.0) = 0.6 - 0.7 = -0.1
totalWeight = 0.6 + 0.7 = 1.3
directScore = -0.1 / 1.3 = -0.0769
```

### Step 2: Rumor Score

Rumor memories: #3 (negative, weight 0.3).

```
weightedSum = 0.3 * -1.0 = -0.3
totalWeight = 0.3
rawScore = -0.3 / 0.3 = -1.0
rumorScore = clamp(-1.0, -0.3, 0.3) = -0.3
```

### Step 3: Values Alignment Score

Check each memory against Brynn's values `["community", "honesty", "family"]`:

- Memory #1 ("helped me repair my shutter"): No strong keyword match for community, honesty, or family. No alignment contribution.
- Memory #2 ("lied about where they found the coin"): Contains "lied" — matches `honesty` (index 1, weight 2x). Valence is negative, so this is negative alignment.
  - Contribution: `-1.0 * 2 * 0.7 = -1.4`, weight: `2 * 0.7 = 1.4`
- Memory #3 ("stole from the merchant"): Contains "merchant" — matches `wealth`. But `wealth` is not in Brynn's values. No match.

```
alignmentSum = -1.4
totalWeight = 1.4
valuesScore = -1.4 / 1.4 = -1.0
```

### Step 4: Soul Modifier

Brynn's tendency "guards their words" does not include "grudge" or "forgiv", so no tendency modifier applies. Her moral grain is `pragmatic`, which has no special rule. Modifier = `0`.

### Step 5: Composite Score

```
opinionScore = clamp(
  (-0.0769 * 0.5) + (-0.3 * 0.2) + (-1.0 * 0.3) + 0,
  -1.0, 1.0
)
= clamp(-0.0385 + -0.06 + -0.3 + 0, -1.0, 1.0)
= clamp(-0.3985, -1.0, 1.0)
= -0.3985
```

### Step 6: Disposition

`-0.3985` falls in the range `>= -0.4`, so the disposition is **wary**.

### Result

Brynn views the player with wariness. The lie about the coin (which violated her core value of honesty) dragged her opinion down significantly, even though the player once helped her with repairs. The rumor about theft reinforced her suspicion. Her pragmatic nature means she does not amplify the negativity further — but she is not quick to forgive either.

```yaml
---
npc_id: "brynn-ashford"
last_updated: "0001-06-01"
opinion_score: -0.3985
disposition: wary
direct_memories: -0.0769
rumors_heard: -0.3
values_alignment: -1.0
memory_modifier: 0
---
```
